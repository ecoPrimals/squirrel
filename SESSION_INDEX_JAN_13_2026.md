## 💡 Key Concepts Explained

### Infant Primal Pattern 🌟

**What**: Each primal starts with zero hardcoded knowledge and discovers its world at runtime.

**Why**: 
- O(1) integration instead of N² hardcoded connections
- Zero vendor lock-in
- True sovereignty
- Ecosystem evolves without code changes

**How**: Use `CapabilityRegistry` to discover services by capability, not by name.

**Example**: See `songbird/mod.rs` lines 84-178

### Capability-Based Discovery

**What**: Discover services by what they can do, not what they're called.

**Example**:
```rust
// ❌ BAD: Hardcoded primal name
let beardog = BearDogClient::new("http://localhost:7443")?;

// ✅ GOOD: Capability discovery
let auth = capability_registry
    .discover_by_capability(&PrimalCapability::Authentication)
    .await?;  // Could be BearDog, could be ANYTHING!
```

### Primal Sovereignty

**What**: Each primal knows only itself and discovers others at runtime.

**Why**: True independence, no compile-time dependencies on other primals.

**How**: `PrimalSelfKnowledge` + `CapabilityRegistry` + `UniversalAdapterV2`
