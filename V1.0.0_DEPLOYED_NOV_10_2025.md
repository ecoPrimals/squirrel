## 🚀 **DEPLOYMENT OPTIONS**

Your v1.0.0 release is now published and ready for production deployment.

### **Option 1: Docker Deployment** 🐳

```bash
cd /home/eastgate/Development/ecoPrimals/squirrel

# Checkout the release tag
git checkout v1.0.0

# Build Docker image
docker build -t squirrel:1.0.0 .

# Tag for your registry
docker tag squirrel:1.0.0 your-registry.com/squirrel:1.0.0
docker tag squirrel:1.0.0 your-registry.com/squirrel:latest

# Push to registry
docker push your-registry.com/squirrel:1.0.0
docker push your-registry.com/squirrel:latest

# Deploy
docker run -d \
  --name squirrel \
  -p 8080:8080 \
  -e SQUIRREL_ENV=production \
  -e RUST_LOG=info \
  --restart unless-stopped \
  your-registry.com/squirrel:1.0.0

# Verify
docker ps | grep squirrel
curl http://localhost:8080/health
```

### **Option 2: Kubernetes Deployment** ☸️

```bash
cd /home/eastgate/Development/ecoPrimals/squirrel

# Checkout release
git checkout v1.0.0

# Build and push Docker image (as above)

# Deploy with Helm
helm upgrade --install squirrel ./deployment/helm/squirrel-chart \
  --namespace production \
  --create-namespace \
  --values values-production.yaml \
  --set image.tag=1.0.0

# Verify
kubectl get pods -n production -l app=squirrel
kubectl logs -f -n production -l app=squirrel

# Check health
kubectl port-forward -n production svc/squirrel 8080:8080 &
curl http://localhost:8080/health
```
