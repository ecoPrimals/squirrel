### **2. Fixed Server Implementation** ✅

**Pattern from Songbird**: Use `LengthDelimitedCodec` as framing layer

```rust
// OLD (broken - direct bincode, no framing):
let transport = tarpc::serde_transport::new(stream, Bincode::default());

// NEW (working - with length-delimited framing):
let transport = tarpc::serde_transport::new(
    LengthDelimitedCodec::builder()
        .max_frame_length(16 * 1024 * 1024) // 16 MB max frame
        .new_framed(stream),
    Bincode::default(),
);
```

**Pattern from Songbird**: Handle stream of responses correctly

```rust
// OLD (broken - tried to await a Stream):
if let Err(e) = channel.execute(server.serve()).await {
    error!("tarpc channel error: {}", e);
}

// NEW (working - for_each on the Stream):
channel
    .execute(server.serve())
    .for_each(|response| async move {
        tokio::spawn(response);
    })
    .await;
```

**Key Discovery**: `serve()` returns a `Stream<Item = Future>`, not a single future.

### **4. Removed Incorrect Attribute** ✅

**Issue**: `#[tarpc::server]` doesn't exist in tarpc 0.34

```rust
// OLD (broken):
#[tarpc::server]
impl SquirrelRpc for SquirrelRpcServer {
    // ...
}

// NEW (working):
// Just implement the trait directly (generated by #[tarpc::service])
impl SquirrelRpc for SquirrelRpcServer {
    // ...
}
```

**Key Discovery**: The `#[tarpc::service]` macro on the trait generates everything needed. No server-side attribute required.

## 📚 **Key Learnings**

### **1. tokio-serde Version Matters**
- tarpc 0.34 requires `tokio-serde 0.8.0` (not 0.9)
- Check reference implementations for exact versions

### **2. Framing is Required**
- Can't use Bincode directly on TCP streams
- Need `LengthDelimitedCodec` to frame messages
- This is the "tarpc-serde" transport pattern

### **3. Streams vs Futures**
- `serve()` returns a Stream of response futures
- Use `.for_each()` to handle each request
- Spawn each response as a task for concurrency

### **4. No Server Attribute**
- `#[tarpc::service]` on trait is sufficient
- No `#[tarpc::server]` attribute exists in tarpc 0.34
- Just implement the generated trait
