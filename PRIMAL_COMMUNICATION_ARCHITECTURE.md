# Primal Communication Architecture

**Date**: January 19, 2026  
**Version**: v1.7.0  
**Question**: Do primals communicate internally by delegating to Songbird?

---

## 🎯 Short Answer

**NO!** Primals do NOT delegate to Songbird for internal communication.

Songbird is **ONLY** used for **external HTTP/network requests** (e.g., calling OpenAI API).

For **primal-to-primal communication**, we use **direct Unix sockets** with:
- **JSON-RPC 2.0** (biomeOS integration)
- **tarpc binary RPC** (high-performance P2P)

---

## 📡 Communication Patterns

### 1. **Primal-to-Primal** (Inter-Primal Communication)

**Pattern**: Direct Unix Sockets + JSON-RPC 2.0

**Examples**:
- Squirrel → BearDog (crypto operations)
- Squirrel → Songbird (HTTP delegation)
- BearDog → Squirrel (capability discovery)

**Implementation**:
```rust
// Direct Unix socket connection
let socket_path = "/run/user/1000/beardog.sock";
let stream = UnixStream::connect(socket_path).await?;

// Send JSON-RPC 2.0 request
let request = json!({
    "jsonrpc": "2.0",
    "method": "crypto.sign",
    "params": { "data": "..." },
    "id": 1
});
```

**Why Direct?**
- ✅ Lowest latency (no intermediary)
- ✅ Simple protocol (JSON-RPC 2.0)
- ✅ biomeOS standard
- ✅ No HTTP overhead

---

### 2. **Squirrel-to-Squirrel** (Peer-to-Peer)

**Pattern**: tarpc Binary RPC (High-Performance)

**Examples**:
- Squirrel Instance A → Squirrel Instance B (federation)
- Squirrel Tower 1 → Squirrel Tower 2 (cross-tower mesh)
- Squirrel Node → Squirrel Coordinator (distributed AI)

**Implementation**:
```rust
// tarpc client (generated by macro)
let addr = "192.168.1.10:9010".parse()?;
let client = connect_tarpc(addr).await?;

// Call remote Squirrel methods
let response = client.query_ai(context::current(), TarpcQueryRequest {
    prompt: "...",
    provider: Some("auto"),
    // ...
}).await?;
```

**Why tarpc?**
- ✅ Binary protocol (faster than JSON)
- ✅ Type-safe (compile-time checks)
- ✅ Proven in Songbird/BearDog
- ✅ Perfect for P2P mesh

---

### 3. **External HTTP** (Network APIs)

**Pattern**: Delegation to Songbird via Unix Socket

**Examples**:
- Squirrel → OpenAI API (via Songbird)
- Squirrel → Anthropic API (via Songbird)
- Squirrel → Any HTTP endpoint (via Songbird)

**Implementation**:
```rust
// Discover Songbird's http.client capability
let songbird_socket = discover_capability("http.client").await?;

// Connect via Unix socket
let stream = UnixStream::connect(songbird_socket).await?;

// Send HTTP request via JSON-RPC
let request = json!({
    "jsonrpc": "2.0",
    "method": "http.post",
    "params": {
        "url": "https://api.openai.com/v1/chat/completions",
        "headers": { "Authorization": "Bearer ..." },
        "body": { "model": "gpt-4", "messages": [...] }
    },
    "id": 1
});
```

**Why Delegate?**
- ✅ Eliminates `reqwest` (and `ring`!)
- ✅ Centralized TLS handling
- ✅ 100% Pure Rust in Squirrel
- ✅ Songbird handles network complexity

## 🔍 Implementation Details

### Unix Socket Discovery

**File**: `crates/main/src/rpc/unix_socket.rs`

**4-Tier Priority**:
1. `SQUIRREL_SOCKET` env var (override)
2. `BIOMEOS_SOCKET_PATH` env var (Neural API)
3. XDG runtime: `/run/user/<uid>/squirrel-<family>.sock`
4. Temp fallback: `/tmp/squirrel-<family>-<node>.sock`

**Example**:
```bash
# Connect to BearDog
export BEARDOG_SOCKET="/run/user/1000/beardog-tower1.sock"

# Connect to Songbird
export SONGBIRD_SOCKET="/run/user/1000/songbird-tower1.sock"

# Start Squirrel (discovers via sockets)
./squirrel
```

---

### JSON-RPC Types

**File**: `crates/main/src/rpc/types.rs`

**Manual Implementation** (Pure Rust!):
```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize, Deserialize)]
pub struct QueryAiRequest {
    pub prompt: String,
    pub provider: Option<String>,
    pub model: Option<String>,
    // ...
}
```

**Why Manual?**
- ✅ NO `jsonrpsee` (has `ring`!)
- ✅ Just `serde_json` (Pure Rust!)
- ✅ Following BearDog's pattern

---

### tarpc Service

**File**: `crates/main/src/rpc/tarpc_service.rs`

**Service Definition**:
```rust
#[tarpc::service]
pub trait SquirrelRpc {
    async fn query_ai(request: TarpcQueryRequest) -> Result<TarpcQueryResponse, String>;
    async fn list_providers() -> Result<Vec<TarpcProviderInfo>, String>;
    async fn health_check() -> Result<TarpcHealthStatus, String>;
    async fn announce_capabilities(capabilities: Vec<String>) -> Result<bool, String>;
    async fn discover_peers() -> Result<Vec<String>, String>;
}
```

**Why tarpc?**
- ✅ Binary protocol (fast!)
- ✅ Type-safe (no JSON parsing errors)
- ✅ Used in Songbird/BearDog (proven)
- ✅ Perfect for P2P mesh

## ✅ Design Principles

### 1. **Capability-Based Discovery**

Primals discover each other via capabilities, NOT hardcoded names:

```rust
// ❌ BAD: Hardcoded
let beardog_socket = "/tmp/beardog.sock";

// ✅ GOOD: Capability discovery
let crypto_service = discover_capability("crypto.sign").await?;
```

---

### 2. **Direct Communication (No Proxy)**

Primals communicate directly, NOT through intermediaries:

```rust
// ✅ GOOD: Direct Unix socket
Squirrel → BearDog (Unix socket)

// ❌ BAD: Unnecessary proxy
Squirrel → Songbird → BearDog (wasteful!)
```

---

### 3. **Delegation ONLY for External**

Songbird is used ONLY for external HTTP:

```rust
// ✅ GOOD: Delegate external HTTP
Squirrel → Songbird → OpenAI API

// ❌ BAD: Delegate internal IPC
Squirrel → Songbird → BearDog (NO!)
```

## 🎯 Key Takeaways

1. **Direct Unix Sockets**: Primals communicate directly via Unix sockets (no proxy!)
2. **JSON-RPC for biomeOS**: Inter-primal uses JSON-RPC 2.0 (manual, Pure Rust!)
3. **tarpc for P2P**: Squirrel-to-Squirrel uses tarpc (binary, high-performance)
4. **Songbird for HTTP**: External APIs delegated to Songbird (TLS handled there!)
5. **Capability Discovery**: NO hardcoded primal names, discover by capability
