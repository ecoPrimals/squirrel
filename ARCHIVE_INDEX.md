## 📚 Why Archive?

### Principle: Documentation as Fossil Record

We keep historical documentation because:

1. **Learning**: See how decisions evolved
2. **Context**: Understand why things are the way they are
3. **Archaeology**: Future developers can dig into history
4. **Patterns**: Identify successful evolution patterns
5. **Mistakes**: Learn from what didn't work

### What We DON'T Archive

- Active reference documentation
- Current version docs
- Operational guides still in use
- Integration templates
- Architecture specifications
